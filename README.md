## 📌 Project Description

### 🔴 Problem

In many local areas, finding reliable service providers like electricians, plumbers, tutors, or delivery agents is difficult. Customers often rely on unverified contacts or word-of-mouth, while skilled workers struggle to find consistent job opportunities.

---

### 🟢 Solution

Our platform provides a **centralized web-based marketplace** where customers can easily discover, compare, and book local services, while service providers can list their skills and connect with nearby clients.

---

### ✨ Key Features

* 🔐 User Authentication (Customer & Provider roles)
* 🔍 Browse & Search Services by Category
* 📅 Easy Booking System with Date & Time selection
* 📊 Dashboard for managing bookings
* ⭐ Rating & Review System
* 📍 Location-based service discovery *(optional)*

## 🚀 Deployment

### Frontend on Vercel

1. Import this repository into Vercel.
2. Keep the project root at the repository root.
3. Add environment variable:
   `VITE_API_URL=https://your-render-backend-url.onrender.com/api`
4. Deploy. The included [vercel.json](/workspaces/Zero2One_Vivek_Maddheshiya/vercel.json) builds the Vite app from `frontend/` and enables SPA routing.

### Backend on Render

1. Create a new Web Service from this repository on Render.
2. Render will detect [render.yaml](/workspaces/Zero2One_Vivek_Maddheshiya/render.yaml).
3. Set these environment variables:
   `MONGODB_URI=your_mongodb_connection_string`
   `JWT_SECRET=your_jwt_secret`
   `FRONTEND_URL=https://your-vercel-frontend-url.vercel.app`
   `CORS_ORIGINS=https://your-vercel-frontend-url.vercel.app`
4. Deploy. The backend health endpoint is `/api/health`.

### Local env files

* Frontend example: [frontend/.env.example](/workspaces/Zero2One_Vivek_Maddheshiya/frontend/.env.example)
* Backend example: [backend/.env.example](/workspaces/Zero2One_Vivek_Maddheshiya/backend/.env.example)
